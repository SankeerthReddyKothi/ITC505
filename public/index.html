ADDENDUM
HTML Code Explanation:
Semantic HTML: The HTML structure is based on semantic elements like <header>, <main>, <footer>, which enhances accessibility and SEO.
Styling: The styling is done using inline CSS within the <style> tags. This keeps the styles scoped to this specific HTML file.
Form: The form is structured with appropriate labels and input fields for collecting noun, verb, adjective, adverb, and place.
Form Submission: The form action is set to /submit-form.php with the POST method. This specifies where the form data will be sent upon submission.
Last Updated: JavaScript code at the end fetches the last modified date of the document and displays it in the footer.

Server.js Code Explanation:
Express Setup: The server is created using Express.js, a Node.js web application framework.
Middleware: express.urlencoded() middleware is used to parse incoming request bodies with URL-encoded payloads. morgan middleware is used for logging HTTP requests.
Static Files: The public directory is served as static files. This includes HTML, CSS, and any client-side JavaScript files.
POST Route Handler: A POST route handler is defined for the /submit-form.php endpoint. It receives the form data, validates it, generates a Mad Lib, and sends back a response with the generated Mad Lib.
Validation: The server checks if all fields (noun, verb, adjective, adverb, place) are filled. If any field is missing, it returns an error message asking the user to fill out all fields.
Response: If all fields are filled, the server generates a Mad Lib using the form data and sends it back as a response to the client.
GET Route Handler: A GET route handler is defined for the root endpoint /. It serves the HTML file for the Mad Lib form.
Port Configuration: The server listens on port 80 by default, but it can be configured to listen on port 8080 locally using the command line argument 'local'.
